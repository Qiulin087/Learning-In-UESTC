# 如何参与贡献

在参与贡献之前，请先确认你想要提交的资料是仓库中没有的，或者是有但已过时需要更新的。

## 贡献方式

> 详细操作步骤请自行询问 ai。若是采用后两种方式，请告知个人 Github 用户名和注册邮箱以确保你能出现在 Contributors 列表中

1. （**推荐**）提 PR：将该仓库 fork 到本地，把你想提交的资料复制到本仓库的对应目录下后发起 pull request，等待维护者合并
2. 提 ISSUE：将想提交的资料整理好打包以附件形式通过本仓库的 [**Issues**](https://github.com/1538-art/Learning-In-UESTC/issues) 界面提交，由维护者来手动添加
3. 发 QQ（1538130391）：将想提交的资料整理好打包发送给维护者，由维护者来手动添加

## 注意事项

- 控制资料体积，体积太大下载和上传都不方便，且单文件体积最多 100M，实际上，超过 50M 就应该考虑精简，或者把 PDF 压缩一下之类的
- 每次 commit 要有一定的细粒度，不要一下上传又多又杂的资料，可以参考前几次 commit 的内容
